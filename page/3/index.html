Loss Function - 损失函数

在课程中的解释如下，其核心在于通过正向传播(Forward Propagation)计算出y^\hat{y}y^​，也就通过神经网络预测的值，之后通过将损失函数不断变小使得y^\hat{y}y^​不断趋近于yyy（即正确的值），损失函数就是单个样例的损失/误差。最直观的损失函数为L(y^,y)=12(y^−y)2L(\hat{y}, y) = \frac{1}{2}(\hat{y} - y)^{2}L(y^​,y)=21​(y^​−y)2来实现，但是这个函数有一个缺陷就是由多个局部最小值，无法判断哪一个最小值是最优解。所以就使用了另一种比较常用的Loss Function: L(y^,y)=−(ylog⁡y^+(1−y)log⁡(1−y^))L(\hat{y}, y) = -(y\log \hat{y} + (1-y)\log (1-\hat{y}))L(y^​,y)=−(ylogy^​+(1−y)log(1−y^​))实现，其原理是：

  • 如果y=1y = 1y=1这个公式就变成了L(y^,y)=−log⁡y^L(\hat{y}, y) = - \log\hat{y}L(y^​,y)=−logy^​，为了使LF变小则需要log⁡y^\log \hat{y}logy^​增大，又因为log⁡x\log xlogx是一个增函数则y^\hat{y}y^​增大；
  • 如果y=0y = 0y=0这个公式就变成了L(y^,y)=−log⁡(1−y^)L(\hat{y}, y) = - \log (1 - \hat{y})L(y^​,y)=−log(1−y^​)，同样为了使LF变小则需要log⁡(1−y^)\log (1 - \hat{y})log(1−y^​)增大，则需要y^\hat{y}y^​减小。

数据库性能优化

数据库两个基本概念

数据库 (Block)

数据块是数据库中数据在磁盘中存储的最小单位，也是一次IO访问的最小单位。一个数据块通常可以存储多条记录，数据块大小是DBA在创建数据库或表空间时指定。一个数据库可以包括多个数据文件，一个数据文件内又包含多个数据块。

RowId

ROWID是每条记录在数据库中的唯一标识。

优化方案

减少数据访问

索引 (Index)

索引表不是万能的，它的存在会大大增加表记录的DML(INSERT,UPDATE,DELETE)开销，正确的索引可以让性能提升100，1000倍以上，不合理的索引也可能会让性能下降100倍。

索引是啥

B-TREE索引也称为平衡树索引(Balance Tree)，它是一种按字段排好序的树形目录结构，主要用于提升查询性能和唯一约束支持。

DEEPLEARNING.AI NOTES FOR LESSON1_WEEK3 QUIZ

Reminder: The general methodology to build a Nerual Network is to:

  1. Define the neural network structure (# of input units, # of hidden units, etc.).
  2. Initialize the model's parameters
  3. Loop:
    • Implement forward propagation
    • Compute loss
    • Implement backward propagation to get the gradients
    • Update parameters(gradient descent)

You often build helper functions to compute steps 1-3 and then merge them into one function wa call nn_model().
